Homework: SQLAlchemy Practice

Introduction

We've created a file model.py that, when run, creates a database of Hackbright employees. The focus of today's homework is to get some practice querying a database using SQLAlchemy to get information that you want. If you need help, take a look at the SQLAlchemy I & II lectures.

Requirements

Before you get started querying, make sure to do the following:

  • Make and run a virtual environment in your working directory.
  • Install the requirements via the requirements.txt file.
  • Run createdb emp in your terminal. (If you get an error, make sure Postgres is running on your computer.)
  • Create and populate the database using python -i model.py.
  • If you run model.py again, it will try to seed the database a second time. PostgreSQL doesn't like this and will throw an error. To avoid this, exit interactive mode, and before running model.py a second time, comment out line 210 (example_data()).

Directions

Now that your database is established, here are the queries that we'd like you to run:

  • First, while running python -i model.py, try using the included functions (all_employees_nav() and all_employees_join()) to get a list of all employees.
  • Next, run server.py, navigate to http://localhost:5000, and check out these functions as routes -- use the Debug Toolbar on the right to note the difference between running times and query numbers for the two functions. Why are they different? What does that mean?
  • Run python -i model.py again, and grab the employee object with the name 'Cynthia Dueltgen.' (Hint: later steps will be easier if you bind the result of this query to a variable name.)
  • What department is she in? (Show the object for her department.)
  • What is the phone number of the department that Cynthia is in?
  • Find all employees whose department we don't know. (Hint: they will have NULL department codes.)
  • Get a list of tuples with just the name of everyone whose favorite color is orange.
  • Find all employees who work in 'Administration.'
